# GitHub 布局、紧凑月历与标题栏时钟修订设计

## 目标

修正当前 GitHub 页面四处空间关系：刷新操作与页面标题对齐；资料卡内的在线状态和打开主页操作互不占位；活动日历改为紧凑的传统月历；系统日期时间移出业务内容并进入应用标题栏。

## 页面标题与刷新操作

GitHub 页面标题区使用左右两端布局。左侧保留 `GITHUB`、`GitHub activity` 和说明文字，刷新按钮固定在同一标题区右侧并垂直居中。刷新状态、禁用状态和现有事件绑定不变，窄窗口下允许标题区换行，但刷新按钮不得覆盖资料卡。

## 资料卡操作分区

资料卡继续横向展示头像、用户名及仓库、粉丝和连续贡献数据。右侧划分为两个明确区域：`Live` 状态位于右上，`Open GitHub profile` 位于右下。两者使用独立容器与间距，不依赖绝对定位互相避让；窄窗口下整体换行为独立操作区。

## 紧凑活动月历

活动日历参考用户提供的传统月历：顶部以较大的月份缩写和强调色年份为主，旁边排列上一月、`TODAY` 和下一月操作；下方是紧凑的星期标题与日期数字网格。相邻月份日期以低对比度显示，当月无贡献日期保持普通文字，有贡献日期用绿色强度或强调色表达贡献量，今天保留清晰但克制的强调。

日历取消大面积圆角日期块，压缩列间距、行高、卡片内边距和整体高度。底部仍保留本月贡献数、活跃天数、最佳单日和连续天数统计，但改为紧凑统计带。月份切换、键盘可达性、提示信息及现有 GitHub 数据结构保持不变。

## 标题栏时钟

主内容区不再渲染日期时间。日期与时间进入应用最上层标题栏，放在 Windows 窗口控制按钮左侧，并避开可拖拽区域和最小化、最大化、关闭按钮。标题栏时钟沿用当前实时更新时间与本地化格式；在窄窗口下可缩短显示内容，但不得重新落回业务页面。

## 验证

- 更新渲染结构与样式契约测试，覆盖刷新按钮、资料卡操作分区、紧凑月历和标题栏时钟。
- 验证刷新、打开 GitHub 主页、月份切换与 `TODAY` 操作仍然可用。
- 运行相关 renderer 测试和完整项目测试。
- 启动 Windows Electron 页面并截图，检查常用窗口尺寸下不存在重叠、裁切或多余空白。
