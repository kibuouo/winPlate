# GitHub 单月活动日历设计

## 目标

- 将 GitHub 活动区重构为一个自主设计的单月翻页日历，而非 GitHub 原版年度热力图或缩小的月度方格。
- 让活动日历作为页面主视觉，充分使用内容宽度，同时保留清晰的日期语义和贡献强度信息。
- 移除日历旁独立的大型月度汇总卡，消除不平衡的左右分栏。

## 页面层级

标题、资料条保持在上方。资料条下方使用全宽 `活动日历` 主卡：卡片顶部为月份名称和上月/下月按钮，中间为 7 列、5 至 6 行的当月日历，底部为简洁的统计带。固定仓库与最近活动维持为主卡下方的两个辅助卡片。

## 日历主卡

- 每个日期为圆角小方块，宽高根据可用宽度在 28px 至 36px 间调整；空白占位日期降低透明度。
- 日期数字置于方块内，贡献强度通过背景色分级；保留键盘焦点和悬停提示。
- 月份切换沿用现有 `data-month-direction` 按钮与禁用逻辑。
- 底部统计带显示本月贡献、活跃天数、最佳单日和连续天数，使用分隔线而非额外卡片。

## 响应式与数据

宽屏日历卡完整占据主内容宽度；窄屏时方块缩小到适合容器的尺寸，统计带换行但仍保持同一卡片。继续使用当前月份的 `counts`、`levels`、`commits` 和连续贡献数据；不修改 GitHub API 或刷新逻辑。
