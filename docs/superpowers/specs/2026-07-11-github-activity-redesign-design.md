# GitHub 活动页重构设计

## 目标

- 消除系统时钟为主内容预留的垂直空间，使所有模块内容从顶部开始排版。
- 将 GitHub 页面重组为清晰的活动概览，而不是一列松散堆叠的资料、仓库与日历卡片。
- 保留贡献热力图，并把它改为紧凑、按周可读的核心信息卡。

## 主内容框架

系统时钟继续在主内容区右上角显示，但脱离普通文档流，不再占用头部高度。`#page-content` 使用更小的顶部内边距，因此各模块标题会向上贴近内容表面。时钟不遮挡页面操作；GitHub 标题行在可用宽度不足时自动为右侧控件留出空间。

## GitHub 页面层级

1. 第一行是页面标题 `GitHub activity`，左侧带简短说明，右侧包含系统时钟和刷新操作。
2. 第二行是约 96px 高的资料条：72px 头像、姓名与用户名、在线状态、仓库/粉丝/连续贡献数据，以及打开 GitHub 主页按钮。
3. 第三行是双列活动概览：左列为贡献热力图，右列为本月汇总。热力图保持 7 行日期语义，但使用固定小方格与紧凑列宽；月份切换和可访问性提示保留。汇总显示本月贡献总数、活跃天数和最高单日贡献。
4. 最后一行放置固定仓库与最近活动，两张同高卡片；在窄屏设备上各区域按阅读顺序纵向排列。

## 数据与交互

不改变 GitHub 数据接口。汇总数据从当前已加载月份的 `counts` 数组计算；缺失数组按零处理。现有刷新按钮、月份切换按钮、贡献单元格提示及 `data-open-github` 主页入口保持可用。

## 验证

- 通过 renderer 源码契约测试覆盖：时钟不再为页面撑高、GitHub 标题优先、热力图仍存在且使用紧凑网格、汇总指标由月份数据生成。
- 执行 Windows Electron renderer 语法检查和完整单元测试。
- 在宽屏与窄屏 CSS 断点下检查布局不会遮挡时钟或压缩热力图。
