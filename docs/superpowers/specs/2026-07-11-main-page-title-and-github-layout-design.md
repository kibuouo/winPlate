# 主界面标题与 GitHub 布局调整设计

## 目标

- 移除主内容区顶部的 `LIVE STATUS` 标识及状态点，取消其占用的垂直空间。
- 保留右上角系统日期和时间，并让各模块的页面标题从内容区顶部开始展示。
- 将 GitHub 详情页从左侧大头像栏改为紧凑的横向资料栏，优先展示贡献与仓库内容。

## 页面框架

主内容区顶部不再渲染状态标题。系统时钟继续显示在右上角；页面内容的首个标题以统一的顶部间距对齐。现有模块的业务数据、刷新逻辑和导航行为均不改变。

## GitHub 详情页

GitHub 页面改为单列内容流：

1. 页面标题和刷新操作位于顶部。
2. 紧凑资料栏紧随其后，使用约 88px 的圆角头像；名称、用户名、仓库数、粉丝数、连续贡献天数、状态和打开主页操作横向排列，并会在窄宽度下自动换行。
3. 固定仓库、贡献日历和贡献活动使用全宽卡片依次展示。

这样避免了 220px 头像和独立左栏挤压主内容宽度，同时保留个人资料识别和所有现有互动入口。

## 响应式与验证

- 宽屏使用横向资料栏；中小屏将资料栏改为可换行或纵向排列，避免压缩日历。
- 现有 GitHub 刷新、月度贡献切换和打开主页按钮保持原有选择器与事件绑定。
- 为渲染结构新增或更新针对 `LIVE STATUS` 移除和 GitHub 紧凑资料栏的测试断言，并运行相关 renderer 测试与项目测试脚本。
