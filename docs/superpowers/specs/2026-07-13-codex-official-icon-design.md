# Codex 官方图标统一设计

## 目标

将 WinPlate 中所有代表 Codex 的通用代码或终端图标，统一替换为官方 Codex 应用图标：蓝紫渐变的云形底座与白色命令提示符。保留现有尺寸、交互与通知圆形容器。

## 范围

- 顶部和模块卡片中的 `codexIcon`。
- 通知页的 Codex 来源图标，以及智能通知中由 Codex 任务推导出的默认图标。
- 其他通过语义键 `codex` 或 Codex 来源渲染的入口。

非目标：不改变通知分类、布局、颜色 token、点击行为或其他来源的图标。

## 设计

1. 在共享的 Electron 图标注册表中新增 `codex` 语义图标，使用本地内联的官方 Codex 风格矢量：蓝紫渐变云形底座与白色命令提示符。
2. 将 `notificationSourceIconKey`、智能通知规则和来源默认值都指向 `codex`，移除 `code` 与 `terminal` 作为 Codex 的回退图形。
3. 模块与详情页复用同一图形定义；其 CSS 仅控制尺寸，不重绘或覆盖标记几何。图标本身保持官方渐变，不随深浅主题反色。
4. 维持现有的 `viewBox`、无障碍隐藏属性和通知圆形背景；该背景只作为图标外侧留白，不覆盖图标的渐变色。

## 验收与测试

- 对 Codex 来源通知和含 “Codex” 的智能通知，断言选择 `codex` 图标键。
- 对模块图标，断言使用官方标记而非通用云代码、代码或终端路径。
- 运行 Electron 包的单元测试与语法检查；不修改当前工作区已有的未提交文件。

## 风险控制

官方标记只作为项目内联 SVG 使用，不引入远程图片或运行时网络依赖。渐变与白色命令提示符固定在图标内部，从而在暗色、浅色与通知底座上均保有官方辨识度。
